## 摘要
大型语言模型（LLMs）越来越多地通过不透明的服务链——API聚合器、转售商和推理提供商——被消费，其中客户端没有技术手段来确认回答的模型是否就是宣传的模型，而最近的审计表明，相当一部分商业端点偏离了供应商的参考权重。现有的识别技术需要长文本生成、token级别的对数概率、对抗性设计的提示或模型所有者的合作。我们证明，更弱的证据就足够了。我们将LLM的行为指纹定义为其对简单单词提示——“在1到100之间随机说一个数字”——的答案的经验分布，跨四种语言收集，每个查询成本为一个输出token。通过测量通过大型商业聚合器（OpenRouter）提供的165个模型，我们发现：(i) 这些分布高度非均匀（中位单元熵1.0比特）且具有模型特异性：同一模型样本的两半分拆之间的距离比不同模型样本之间的距离小一个数量级；(ii) 指纹之间的Jensen-Shannon散度可恢复模型谱系，以59.5%的留一法准确率（随机概率为18.4%）将模型归入其文档化的家族；(iii) 一种生物识别风格的验证协议在使用完整40单元电池时达到7.3%的等错误率，在使用八个探测单元时低于11%——每次审计大约一百次单token查询。我们进一步报告了生态系统异常，包括一个专有品牌旗舰端点与开源Qwen模型在分布上无法区分。该协议、提示、原始数据和分析代码均已发布，以供复现和操作使用。

## Abstract
Large language models (LLMs) are increasingly consumed through opaque serving chains - API aggregators, resellers, and inference providers - in which the client has no technical means to confirm that the model answering is the model advertised, and recent audits show that a substantial fraction of commercial endpoints deviate from the vendor's reference weights. Existing identification techniques require long generated texts, token-level log-probabilities, adversarially crafted prompts, or the model owner's cooperation. We show that far weaker evidence suffices. We define a behavioral fingerprint of an LLM as the empirical distribution of its answers to trivial one-word prompts - "name a random number between 1 and 100" - collected across four languages at a cost of one output token per query. Measuring 165 models served via a large commercial aggregator (OpenRouter), we find that (i) these distributions are highly non-uniform (median cell entropy 1.0 bit) and model-specific: split halves of the same model's samples lie an order of magnitude closer than samples of different models; (ii) Jensen-Shannon divergence between fingerprints recovers model lineage, assigning a model to its documented family with 59.5% leave-one-out accuracy against an 18.4% chance rate; and (iii) a biometric-style verification protocol achieves a 7.3% equal error rate with the full 40-cell battery, and below 11% with eight probe cells - roughly a hundred single-token queries per audit. We further report ecosystem anomalies, including a proprietary-branded flagship endpoint distributionally indistinguishable from an open-weight Qwen model. The protocol, prompts, raw data, and analysis code are released for reproduction and operational use.

- **核心问题**：能否以极低成本（每个查询仅一个输出 token）、无需合作、对普通流量不可区分的方式，验证所服务的模型身份？

## 2. 方法论

### 核心思想
- LLM 在面对“随机数”等简单提示时，其单 token 输出分布是非均匀且模型特异的。这种分布是模型 tokenizer、训练数据先验、指令微调等层级的综合产物，可作为**行为指纹**。
- 提出**指纹定义**：对于模型 M，在各任务-语言对（称为 probe cell）上收集 n 个采样，得到经验分布 \(\hat{p}_{M}^{t,\ell}\)。所有 probe cell 的分布集合构成 M 的指纹。
- 使用**Jensen–Shannon 散度（JSD，以 2 为底）** 测量指纹间距离：\(D(M_a, M_b) = \frac{1}{|B'|} \sum_{(t,\ell) \in B'} \text{JSD}(\hat{p}_{M_a}^{t,\ell} \parallel \hat{p}_{M_b}^{t,\ell})\)，其中 B' 是双方至少各有 10 个有效样本的细胞子集。

### 关键技术细节
- **probe battery**：10 个任务 × 4 种语言（英、俄、中、阿）= 40 个细胞。任务包括：1-100 随机数、1-10 随机数、最喜欢的数字、随机字母、随机单词、随机颜色、最喜欢的颜色、随机动物、随机城市、掷硬币。
- **采样参数**：温度 1.0，固定系统提示强制单词回答，硬上限 16 个 completion token，禁用推理模式。每个细胞采样 30 次（前沿模型 15 次），另加温度 0 采样 3 次。
- **数据规范化**：Unicode NFC、去除标点、大小写折叠、数字系统映射、颜色词规范化等。

### 算法流程（文字描述）
1. **注册阶段**：从可信参考部署收集某个宣称模型 X 的指纹（40 个细胞各自分布），记作 \(\hat{p}_{\text{ref}}\)。
2. **审计阶段**：从待测端点 E 收集同样 probe cells 的 n 个采样，得到 \(\hat{p}_{E}\)。
3. **计算距离**：\(s = \frac{1}{|B'|} \sum \text{JSD}(\hat{p}_{\text{ref}} \parallel \hat{p}_{E})\)。
4. **决策**：若 \(s \le \tau\)（阈值），则接受身份声明；否则拒绝。

对于谱系恢复（RQ2），使用该距离矩阵进行层次聚类（UPGMA）和留一法 1-NN 分类。

## 6. 主要结论与发现

1. **指纹存在且稳定**：单 token 输出分布高度非均匀（中位 cell 熵 1.0 bit），同模型拆分半 JSD 中位数 0.075，不同模型间 0.489。跨提供商时，70.6% 的提供商对距离仍低于 impostor 5% 分位。
2. **谱系恢复有效**：留一法 1-NN 分类准确率 59.5%（随机 18.4%，提高 3.2 倍）。主要家族（GLM, Mistral, GPT）精度/召回较好。
3. **验证可行**：全 40 细胞电池 AUC=0.971，EER=7.3%；8 个细胞时 EER=10.6%，240 个查询即可达到，成本极低。
4. **生态系统异常**：
   - writer/palmyra-x5 的指纹与 qwen/qwen3-235b-a22b-2507 距离 0.141，位于同模型距离中值 0.140，暗示可能被替代。
   - 29% 的同模型不同提供商对指纹距离超出 impostor 范围，表明服务差异化。
   - 发现隐蔽推理阶段、提供商忽略 reasoning-disable 标志等异常。

## 7. 优点

- **极端轻量**：每个查询仅需一个输出 token，无需 logits、无需长生成、无需合作。

- **鲁棒性好**：对提示改写（open-ended paraphrase family）天然鲁棒，因指纹基于分布而非特定字符串。
- **无需白盒**：完全黑盒操作，只依赖文本回复。
